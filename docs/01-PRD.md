
# MCP-CLI 项目需求文档（PRD）

## 1. 项目概述

### 1.1 背景与问题
MCP (Model Context Protocol) 作为一种标准化的模型上下文交互协议，已经涌现出大量不同的服务端实现。然而，在开发、调试和编排这些服务时，开发者与 AI 系统面临几个痛点：
- 每个 MCP 服务的调用方式差异大，缺乏统一的命令行交互入口。
- 工具参数由服务端动态定义，人工查看或编写 CLI 参数容易出错。
- 多个 MCP 服务的连接配置分散，无法集中管理。
- 需要频繁为 AI 智能体（Skill）更新可用工具的文档，手动维护成本极高。
- 智能体配置MCP会占用大量的Token，而Skill使用命令可以降低Token占用。

### 1.2 项目目标
构建一个**通用的 CLI 代理工具 `mcp-cli`**，它能够：
1. 通过统一配置文件兼容 Claude Code 的 MCP 服务器配置格式，集中管理多个异构 MCP 服务。
2. 动态发现并暴露任意 MCP 服务的工具、资源和提示为命令行子命令，并自动解析参数。
3. 为每个 MCP 服务自动生成独立、结构化的 Markdown 使用指南，供 AI Skill 直接引用。

### 1.3 适用范围
- 面向使用 MCP 协议进行工具集成和 AI 工作流编排的开发者和 AI 工程师。
- 适用于本地开发调试、CI/CD 管道中的自动化测试、以及作为 AI 智能体的可靠工具中间层。

## 2. 用户角色与场景

| 角色 | 描述 |
|------|------|
| 开发者 | 需要快速调试 MCP 服务、查看能力清单、执行工具调用 |
| DevOps/集成工程师 | 需要在脚本中调用统一 CLI 接口，组合多个 MCP 服务 |
| AI Skill 设计者 | 需要获得最新的工具使用文档，指导模型生成正确的命令 |

**典型场景**：
- 开发者配置了 `weather` 和 `filesystem` 两个 MCP 服务，想通过 `mcp-cli --server weather get_weather --city Beijing` 直接获取天气。
- 执行 `mcp-cli --config mcp.json --server weather guide` 生成 `weather.md` 文档，用于 AI 技能上下文。
- 脚本中批量调用多个服务，结果以 JSON 输出供下游处理。

## 3. 功能需求

### FR-1 配置文件管理
- 支持读取符合 Claude Code 规范的 JSON 配置文件（含 `mcpServers` 字段）。
- 配置项至少包含：服务名、传输类型（`stdio`/`sse`/`http `）、启动命令/URL、环境变量、请求头等。
- 若未指定 `--server`，程序应列出配置中所有可用服务名和简单描述。

### FR-2 多传输支持
- 支持 `stdio` 模式：根据配置中的 `command` 和 `args` 启动子进程，通过标准输入/输出通信。
- 支持 `sse` 模式：连接到指定的 HTTP SSE 端点，支持自定义 `headers`。
- 支持 `http` 模式：连接到指定的 HTTP 服务器url，支持自定义 `headers`和`oauth`。

### FR-3 动态能力发现
- 连接成功后，自动获取服务端的工具列表 (`list_tools`)。
- 也可获取资源列表和提示列表（扩展目标）。

### FR-4 动态命令行构建
- 根据获取到的工具清单，动态注册为子命令（工具名即为子命令）。
- 根据每个工具的 `input_schema` 自动生成 CLI 参数：
  - 字符串、整数、数字、布尔类型分别映射为对应 argparse 参数。
  - 枚举约束映射为 `choices`。
  - 必填参数标记 `required=True`。
  - 有默认值时使用 `default`。
  - 复杂类型（对象、数组）提供 JSON 字符串透传选项。
- 保留内置命令 `list`（列出工具）和 `guide`（生成指南）。

### FR-5 工具调用
- 执行 `mcp-cli --server mcp_server_name <tool_name> [arguments...]` 时，解析参数并调用 `call_tool`。
- 输出结果支持 JSON 格式（结构化数据）和可读文本（人类友好格式）。
- 调用错误时提供清晰错误信息和非零退出码。

### FR-6 自动生成 Markdown 指南
- 执行 `guide` 命令时，为该 MCP 服务生成独立的 Markdown 文件。
- 文件名格式：`<server_name>.md`，默认输出到 `./mcp-docs/` 目录（可配置）。
- 生成内容包含：
  - 服务名和简介。
  - 每个工具的名称、描述。
  - 参数表格（参数名、类型、必填/可选、默认值、说明）。
  - 命令行使用示例。
- 支持 `--all` 选项来为配置文件中所有服务生成指南。

### FR-7 交互与帮助
- 每一层子命令均支持 `--help` 显示用法和参数说明。
- 提供全局命令概览：`mcp-cli --help`。

## 4. 非功能需求

| 需求 | 说明 |
|------|------|
| 可移植性 | 纯 Python 实现，跨平台运行（Linux/macOS/Windows），依赖清晰。 |
| 易用性 | 安装后单个命令即可使用，无需编写额外代码。 |
| 性能 | 动态生成部分应在 1 秒内完成，调用延迟主要取决于服务本身。 |
| 扩展性 | 新增 MCP 传输类型或配置格式时，架构支持插件式扩展。 |
| 兼容性 | 配置文件100%兼容 Claude Code 现有 MCP 服务器定义。 |
| 文档自更新 | 生成的 Markdown 须与实时服务能力严格一致，零手动维护。 |

## 5. 约束与假设
- 依赖库：`fastmcp` 提供 MCP 客户端能力，`argparse` 为标准库，使用 `asyncio` 异步模型。
- 假设所有目标 MCP 服务都正确实现了 `tools/list` 和 `tools/call` 协议方法。
- 复杂嵌套对象参数建议优先使用 JSON 字符串透传，扁平化映射为可选增强。
- 生成的 Markdown 指南定位为 AI Skill 的专业参考，不包含过于华丽的样式。

## 6. 验收标准
- 使用标准 Claude Code 配置文件启动，能够列出工具并成功调用任意可用的 MCP 工具。
- 生成的 Markdown 指南与实际工具参数完全一致，无遗漏、无错误。
- `--help` 输出清晰完整，参数说明与 `input_schema` 的 `description` 对应。
- 在不指定 `--server` 时，程序友好提示并退出。
